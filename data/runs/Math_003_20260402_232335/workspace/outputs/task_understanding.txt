Task objective: Build an autonomous research workflow for olympiad-level Euclidean geometry theorem proving, using the workspace benchmark to study how formal problem statements can be turned into machine-verifiable, human-readable proofs. The immediate goal is to analyze the provided benchmark/problem language and prepare a research pipeline that can evaluate or prototype proof-oriented methods on these geometry instances.

Available data and inputs: The main benchmark file is data/imo_ag_30.txt, which contains 30 translated IMO geometry problems written in a formal construction language with hypotheses and target predicates (for example cong, cyclic, coll, perp, para, eqangle). Supporting schema files are data/defs.txt, which defines construction primitives such as angle_bisector, foot, incenter2, circle, etc., and data/rules.txt, which lists geometric inference rules and theorem schemas useful for symbolic reasoning. Related-work inputs include four PDFs in related_work/; from quick inspection they contain broad background on Transformers, neural theorem proving with language models, and search-based reasoning, though not all are geometry-specific.

Main planned analysis steps: (1) parse and characterize the benchmark language, constructions, and goal types in imo_ag_30.txt; (2) summarize the primitive definitions and inference rules from defs.txt and rules.txt to understand the available symbolic vocabulary; (3) design and implement reproducible analysis code in code/ to extract dataset statistics, representation patterns, and proof-relevant structure; (4) generate figures showing problem distribution, construction/goal frequencies, and other benchmark properties; (5) formulate and discuss a research direction for autonomous geometry proving, grounded in the benchmark structure and informed by the related work; and (6) write a report that explains methodology, results, validation/limitations, and implications for neuro-symbolic theorem proving.

Expected deliverables: analysis scripts in code/; intermediate parsed/statistical outputs in outputs/; at least several saved figures in report/images/ referenced from the final report; and a publication-style report at report/report.md covering task context, benchmark/data overview, methodology, main findings, comparison to related ideas, and limitations.
